Flight Path Calculation

Flight Path Calculation

April 2020

Technologies: Java, Data Structuresm Object Orineted Programming

The objective of this project was to find the three best possible flight paths between two requested locations based on the different flights given as the data. The best way to complete the objective would be to create an Adjacency List/Graph data structure that holds information about each of the locations and their respective flights. In this project, I heavily utilized Object Oriented Programming. The first object is ‚ÄúCity‚Äù which represents a point in the adjacency list and has the location name, cost, and time to get there as its instance variables. This object is NOT unique for each location as a location can have multiple flights going to it. The second object is ‚ÄúCities‚Äù which represents one list within the overarching adjacency list/graph data structure and gets instantiated with the city name and a Linked List of the ‚ÄúCity‚Äù object, which was created earlier. This object is unique to each given location. Each ‚ÄúCities‚Äù is one location and its Linked List contains all the possible locations it can travel to directly. The next object is ‚ÄúAdjacencyList‚Äù which is a Linked List of the ‚ÄúCities‚Äù object. The Adjacency List is created by first making a ‚ÄúCities‚Äù instance for each unique location provided, and then adding all the possible locations that ‚ÄúCities‚Äù can travel to into its specific ‚ÄúCity‚Äù Linked List. I made an object that will later help with backtracking called ‚ÄúTheStack‚Äù which is just an implementation of a Stack using a Linked List of Linked Lists of Strings. The ‚ÄúFlightCalculation‚Äù object does all the computing for finding the three most efficient paths between two locations. First, using iterative backtracking (utilizing ‚ÄúTheStack‚Äù object), ‚ÄúFlightCalculation‚Äù gets all possible paths between the requested source and destination. I then order these paths based on cost or time using a Tree Set and output the first three paths into a text file.

The first screenshot shows the requested departure and arrival locations and if the paths should be ordered based on time(T) or cost(C). The second image shows all the flights avaialble and the distance/cost of each flight. The last image shows the three best possible paths based on the requested text file.
