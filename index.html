Dash Shell

Dash Shell

January 2021

Technologies: C

In this project, I implemented a fully functional bash shell using C. I used C’s fork() method to create child processes for inputted user commands. To make the shell versatile, I implemented two modes: interactive mode (which took inputs straight from the user) and batch mode (which took inputs from a text file). I also integrated built-in commands including cd, exit (to exit the shell), and path (to set paths from which user commands are executed). I utilized getline() and strtok() functions to receive and parse user input. I allowed the execution of multiple commands in parallel using the ‘&’ symbol and output redirection to a file using the ‘>’ symbol. In case of any errors, whether it is an incorrect command or a failed fork, the program will write a custom error message to the standard error. I carefully handled dynamic and static memory allocation to ensure no memory leaks occurred.

Subsort

Blogs For All

Blogs For All

July 2020

Technologies: Django, Python, PostgreSQL, HTML/CSS, Bootstrap 4, Javascript, Ajax, Heroku, AWS

Despite having “Blogs” in the name, this is more of a Social Media website. In this web application, users can post blogs publicly or keep it in the drafts for a later date, like and comment on their favorite blogs (and of course reply to other comments), and follow their favorite users. Users can also personalize their profile by adding their bio, some of their interests, and other social links. Also, users receive database notifications if another user interacts with their blogs or profile (liking/commenting on a blog or a new follower).

Making the reply system to the comments was difficult because a comment can have many nested replies, but it’s impossible to make a model for every nested comment. However, I was able to get around this issue by making the Comment model recursively related to itself. At first, I didn’t know how to handle the follower/following system, but I solved this problem by making an intermediary model that has 2 one to many relationships, the follower and the followee.

I built the backend for this website using the Django MVC framework and a PostgreSQL database and the frontend using a Bootstrap 4 template with plain HTML/CSS. With Django’s prebuilt security features, I ensured that guest users would have limited access to the website and logged in users won’t be able to update or edit any other user’s blogs or profiles. I used Javascript and Ajax to avoid reloading some form submissions, such as liking a post, to make the website more user friendly. I deployed the website using Heroku. Since Heroku doesn’t store images on an application for long periods of time, I used AWS S3 buckets to handle static files (images and CSS/Javascript files) and media files (user’s profile picture and blog images).

Flight Path Calculation

Flight Path Calculation

April 2020

Technologies: Java, Data Structuresm Object Orineted Programming

The objective of this project was to find the three best possible flight paths between two requested locations based on the different flights given as the data. The best way to complete the objective would be to create an Adjacency List/Graph data structure that holds information about each of the locations and their respective flights. In this project, I heavily utilized Object Oriented Programming. The first object is “City” which represents a point in the adjacency list and has the location name, cost, and time to get there as its instance variables. This object is NOT unique for each location as a location can have multiple flights going to it. The second object is “Cities” which represents one list within the overarching adjacency list/graph data structure and gets instantiated with the city name and a Linked List of the “City” object, which was created earlier. This object is unique to each given location. Each “Cities” is one location and its Linked List contains all the possible locations it can travel to directly. The next object is “AdjacencyList” which is a Linked List of the “Cities” object. The Adjacency List is created by first making a “Cities” instance for each unique location provided, and then adding all the possible locations that “Cities” can travel to into its specific “City” Linked List. I made an object that will later help with backtracking called “TheStack” which is just an implementation of a Stack using a Linked List of Linked Lists of Strings. The “FlightCalculation” object does all the computing for finding the three most efficient paths between two locations. First, using iterative backtracking (utilizing “TheStack” object), “FlightCalculation” gets all possible paths between the requested source and destination. I then order these paths based on cost or time using a Tree Set and output the first three paths into a text file.

The second image shows all the flights avaialble and the distance/cost of each flight. The last image shows the three best possible paths based on the requested text file.
